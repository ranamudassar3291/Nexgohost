PROJECT:
Add automation and domain pricing management to a WHMCS-like hosting billing panel.

--------------------------------------------------

1. CRON BILLING AUTOMATION

Create a cron job that runs every 5 minutes.

Cron tasks:

• Check all services
• Check due dates
• Generate invoices automatically

Logic:

If today = service.next_due_date
Generate invoice.

Invoice amount = package price

Update service.next_due_date according to billing cycle.

Example:

Monthly → +30 days
Yearly → +365 days

Cron command example:

*/5 * * * * php /var/www/billing/cron.php

Database tables used:

services
invoices
clients
packages

--------------------------------------------------

2. AUTO SUSPEND OVERDUE HOSTING

Cron must check unpaid invoices.

If:

invoice.status = unpaid
AND
invoice.due_date + 3 days passed

Then:

Suspend hosting account automatically.

Use module API.

For cPanel / WHM:

WHM API

suspendacct

Parameters:

username
reason = "Overdue Invoice"

Update service status:

status = Suspended

--------------------------------------------------

3. AUTO DOMAIN RENEWAL

Cron must check domain expiry.

If domain expires in 7 days:

Check client credit or auto-renew enabled.

If auto-renew = ON

Then:

Call registrar API to renew domain.

Update:

domain expiry date
create renewal invoice if needed.

Database table:

domains

fields:

expiry_date
auto_renew
status

--------------------------------------------------

4. EMAIL INVOICE REMINDERS

System must send automatic emails.

Schedule:

7 days before due date
3 days before due date
on due date
3 days overdue

Email templates:

Invoice Reminder
Invoice Overdue Notice

Use SMTP mail system.

Database:

email_logs

fields:

client_id

sent_at

5. FRAUD PROTECTION

When new order is placed:

Run fraud checks.

Checks:

IP location
Multiple orders from same IP
Disposable email detection

If risk score high:

Order status = Fraud

Admin must review before activation.

Create table:

fraud_logs

fields:

order_id
ip_address
risk_score
status
